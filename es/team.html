Equipo

Al igual que el resto del proyecto Processing, Processing para Android es un esfuerzo colectivo. El trabajo en el mismo comenzó en el año 2010, cuando Ben Fry y Jonathan Feinberg crearon las componentes básicas del modo Android y la biblioteca principal, y Andres Colubri implementando el motor de gráficos de OpenGL ES. Desde entonces, hemos recibido contribuciones de muchas personas:

  • Google Summer of Code (2022): Rupesh Kumar completó la actualización del modo Android para Processing 4.
  • Google Summer of Code (2020): Aditya Rana implementó un modo experimental que soporta el lenguaje Kotlin para desarrollo en Android.
  • Google Summer of Code (2019): Deeraj Esvar realizó mejoras en el acutalizador del SDK y en el soporte del emulator.
  • Google Summer of Code (2018): Syam Sundar Kirubakaran implementó la nueva biblioteca AR y Manav Jain portó el depurador del Java mode al modo Android.
  • Google Summer of Code (2017): Rupak Das agregó soporte para Gradle Build e implementó el nuevo instalador del SDK, mientras que Sara Di Bartolomeo , creó un proyecto de visualización de audio en VR utilizando la nueva biblioteca de VR disponible en el modo Android.
  • Google Creative Lab New York (2016): Con el apoyo de Creative Lab, Andrés extendió el modo para incluir fondos de pantalla animados, caras de relojes, y aplicaciones en VR para cardboard, además de crear este sitio.
  • Google Summer of Code (2015): Umair Khan actualizó el modo para que sea compatible con Processingo 3, refactorizó la biblioteca principal en base a Fragmentos, y escribió una biblioteca de captura y reproducción de video.
  • Google Summer of Code (2014): Imil Ziyaztdinov implementó la funcionalidad de exportar de paquetes firmados, selección de dispositivos, y descarga automática del SDK.

Este posteo en Medium descrive de manera breve la historia Processing para Android. Para acceder a una lista completa de personas que han contribuido al proyecto, mira esta página.
